# NetGuard AI — Real-Time Network Intrusion Detection System

A machine learning powered Intrusion Detection System (IDS) that streams NSL-KDD traffic records, detects malicious activity, classifies attack types, and displays live results on a Flask + SocketIO dashboard.

---

## Features

- Real-time traffic simulation using the NSL-KDD **test** set (unseen data)
- Random Forest classifier with live confidence scores and SHAP explanations
- Attack categories: **DoS, Probe, R2L, U2R** — and Normal
- **Ground truth validation** — Correct Detection, False Positive, False Negative, Misclassification
- **Severity levels** — Critical / High / Medium / Low based on model confidence
- **Live metrics** — Precision, Recall, F1, FPR, FNR updating every record
- **Attack timeline** — live line chart of attacks vs normal per minute
- **Multi-model comparison** — Random Forest, XGBoost, Decision Tree, Linear SVM
- **Threat Intelligence** — per-attack descriptions, vector, impact, and mitigation in the detail drawer
- **Session management** — each run tracked separately in SQLite
- **Login / Authentication** — CSRF-protected login page, configurable credentials, session expiry
- PDF report generation with confusion matrix, feature importance, and SHAP charts
- CSV and JSON export of detection history
- Docker support — single command deployment

## Quick Start — Local

### 1. Download the Dataset

Get `KDDTrain+.txt` and `KDDTest+.txt` from the Canadian Institute for Cybersecurity:
<https://www.unb.ca/cic/datasets/nsl.html>

Place both files in `data/`:

```text
data/KDDTrain+.txt
data/KDDTest+.txt
```

### 2. Install Dependencies

```bash
pip install -r requirements.txt
```

### 3. Train the Model

Run once before starting the server. Generates all `.pkl` artifacts and chart images.

```bash
python train.py
```

Expected output:

```
✅ Accuracy : 86.03%
✅ Training complete! You can now run: python app.py
```

### 4. Run the Dashboard

```bash
python app.py
```

Open your browser at:

```
http://127.0.0.1:5000
```

Log in with the default credentials (username: `admin`, password: `netguard123`).

## Quick Start — Docker

Docker handles all dependencies automatically. No `pip install` needed.

### Prerequisites

- Docker Desktop installed and running
- Dataset files in `data/`
- Trained model artifacts in `models/` and charts in `static/reports/`
  *(run `python train.py` on your host first, or use the trainer service below)*

### Option A — Run the pre-trained app

```bash
docker-compose up
```

Open: <http://localhost:5000>

### Option B — Train inside Docker, then run

```bash
# Train the model inside a temporary container
docker-compose run --rm trainer

# Then start the dashboard
docker-compose up ids
```

### Common Docker commands

```bash
# Build the image
docker-compose up --build

# Run in background
docker-compose up -d

# View live logs
docker-compose logs -f ids

# Open a shell inside the running container
docker-compose exec ids bash

# Stop everything
docker-compose down
```

## Authentication

The dashboard is protected by a login page. Credentials are set via environment variables.

| Variable | Default | Notes |
|---|---|---|
| `IDS_USERNAME` | `admin` | Dashboard login username |
| `IDS_PASSWORD` | `netguard123` | Change this before any public deployment |
| `IDS_SHOW_LOGIN_HINT` | `0` | Set to `1` to show default credentials on the login page |

To change credentials without editing code:

**Local:**
```bash
IDS_USERNAME=analyst IDS_PASSWORD=mypassword python app.py
```

**Docker:**

Create a `.env` file in the project root:
```env
IDS_USERNAME=analyst
IDS_PASSWORD=mypassword
IDS_SECRET_KEY=a-long-random-secret-string
```

Docker Compose picks it up automatically.

## Model Performance

Evaluated on the NSL-KDD test set (22,544 samples, unseen during training):

| Model | Accuracy | Precision | Recall | F1 |
|---|---:|---:|---:|---:|
| **Random Forest** | **86.03%** | 82.32% | 86.03% | **81.77%** |
| XGBoost | 85.44% | 80.09% | 85.44% | 81.35% |
| Linear SVM | 82.88% | 76.15% | 82.88% | 78.75% |
| Decision Tree | 81.43% | 83.84% | 81.43% | 78.35% |

Random Forest is used as the live streaming model. All four models are shown in the **Models** tab of the dashboard.

### Attack Categories Detected

| Category | Description | Examples |
|---|---|---|
| DoS | Denial of Service | neptune, smurf, back, pod, teardrop |
| Probe | Network Scanning | ipsweep, nmap, portsweep, satan |
| R2L | Remote to Local | guess_passwd, ftp_write, imap, multihop |
| U2R | User to Root | buffer_overflow, rootkit, loadmodule, perl |

## Dashboard Overview

| Tab | Contents |
|---|---|
| **Overview** | Live counters, severity breakdown, precision/recall/F1, attack timeline chart, donut chart |
| **Live Feed** | Scrolling table of every streamed record with severity badge and validation status |
| **History** | Searchable, filterable, paginated database of all past detections with CSV/JSON export |
| **Models** | Multi-model comparison table and bar chart, feature importance, SHAP summary |
| **Reports** | Download full PDF report; view confusion matrix, feature importance, SHAP charts |

Clicking any row opens a **detail drawer** showing:
- Predicted label, category, confidence, severity
- Ground truth validation result (Correct Detection / False Positive / False Negative / Misclassification)
- Traffic features (protocol, service, src bytes, duration)
- SHAP feature contribution bars
- Threat Intelligence profile (attack description, vector, impact, mitigation)

## Notes

- Run `python train.py` before `python app.py`. The app will not start without the model artifacts in `models/`.
- `logs/ids_database.db` and `logs/alerts.log` are runtime files. Delete them to reset all history.
- History and Reports default to the **current session**. Past sessions are stored and queryable.
- The Scapy sniffer (`scripts/sniffer.py`) is a stub. It captures packets but does not yet route them through the model — live packet capture is a planned future feature.
- SQLite runs in WAL mode to support concurrent read/write from the stream thread and Flask API simultaneously.
